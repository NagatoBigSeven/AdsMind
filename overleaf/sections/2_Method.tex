
\subsection{Overview of AdsMind}
%The defined problem that AdsMind provides solution
%化学搜索空间中，指导AdsMind寻找最低能量构型。

To identify the most stable adsorption configuration and energy of a given adsorbate--surface system, AdsMind implements a closed-loop agent architecture composed of four core execution modules: \textbf{Planner}, \textbf{Validator}, \textbf{Executor}, and \textbf{Analyzer}. In each iteration, a structured adsorption hypothesis is proposed by the \textbf{Planner}, checked by the \textbf{Validator}, instantiated and relaxed by the \textbf{Executor}, and interpreted by the \textbf{Analyzer} through post-relaxation structure analysis.
% 不单独解释final analyzer模块，每一轮的后弛豫analyzer和executor一起介绍
A separate \textbf{Summarizer} acts as the report-and-routing node after each physical execution or explicit termination. It either returns control to the planner when the search should continue, or emits the final report of the best configuration once the terminate action occurs or the budget is exhausted. The following sections explain the pipeline and modules in detail.

\subsection{Planner and Validator: Adsorption Hypothesis Generation}


%% Planner

At the beginning of the analysis, the user provides a request prompt together with the corresponding adsorbate SMILES string and surface slab file.
% 这里可以解释对于一个具体体系是如何预处理SMILE和slab文件的，分析表面文件。
At the onset of each iteration, the planner proposes a structured adsorption hypothesis in JSON format, including the canonical site type (\textit{ontop}, \textit{bridge}, or \textit{hollow}), the intended surface symbols, and the adsorbate binding indices.
% 对于planner都需要什么样的context，下文已经解释的足够清楚，此处应该提及闭环结构稍微解释一下context的内容。
% 在整个loop中，输入planner的上下文除了体系的信息、用户的询问，还有一些别的模块写入的上下文，在此处介绍上中包含的信息都有什么，重点在于我们的loop是如何迭代的idea。

In our closed-loop architecture, the updated context from other modules is entered into the planner prompt, providing physical feedback and performance analysis from previous hypotheses.
The planner prompt is assembled from conditional sections rather than a monolithic, fixed template. Three runtime switches control the agentic feedback exposed to the language model. 
\textit{Slip feedback} determines whether prior slips are framed as explicit evidence that the originally planned site is unstable. 
\textit{FORBID} determines whether slipped or otherwise failed site intents are converted into an explicit negative constraint in the planner history. 
\textit{Termination} determines whether convergence-to-known-best signals are surfaced to the planner and whether the planner is allowed to stop early after repeated convergence.

%% Validator
After the hypothesis is proposed by the \textbf{Planner}, the \textbf{Validator} will subsequently enforce chemistry- and schema-level constraints before any expensive physics simulation is made. The validation protocol ensures that:
\begin{enumerate}
    \item The requested adsorbate binding indices are valid and correspond to non-hydrogen heavy atoms, unless the adsorbate is explicitly a single atom (e.g., [H]).
    \item The cardinality of the binding indices matches the topological requirements of the specified site type (e.g., \textit{ontop} requires exactly one index, while \textit{bridge} allows up to two indices for side-on modes).
    \item The hypothesis has not been previously evaluated within the current search trajectory, thereby preventing redundant calculations.
\end{enumerate}
Hypotheses failing these checks elicit an immediate penalty feedback prompt, directing the LLM to refine its proposal without advancing the state machine to the physical simulation phase.

\subsection{Executor and Analyzer: Physical Simulation and Scientific Analysis}

%% Executor
With the validated adsorption hypothesis, we perform physical simulation to evaluate the adsorption energy and performance. 
% 这部分更多是一个整体介绍，需要看看怎么和后文合并。
In AdsMind, the tool executor instantiates the accepted plan on the slab, performs physical relaxation, and returns a structured post-relaxation analysis rather than only a scalar energy. % 2-stage evaluation
The analyzer records whether the final structure remained bound, whether dissociation or rearrangement occurred, and whether the realized coordination environment differs from the planner's intent. These deviations are logged as slip diagnostics. The code-level \textit{Chemical Slip} flag is reserved for cases where relaxation realizes a different surface-symbol fingerprint than the planner requested; canonical site-family mismatch is retained separately as a geometric/site slip signal in the search history.


% 这部分具体介绍了在MLFF弛豫前，对吸附物从SMILE到位点几何的处理。
Before the physical simulation, the proposal is pre-processed by being translated into an atomistic starting structure through a surrogate-SMILES geometry layer. 
This layer maps the selected binding atom(s) to proxy adsorption markers, constructs physically valid initial placements for \textit{ontop}, \textit{bridge}, and \textit{hollow} adsorption, and samples a capped set of candidate surface sites and conformers. In the benchmark configuration used here, the default cap is eight filtered surface coordinates matching the requested site type and surface-symbol pattern and four conformers per site, with one top-ranked candidate carried into full relaxation.

% 这部分介绍了具体的candidate选择和计算，以及本轮报告生成。衔接到上面的段落中即可。
A two-step evaluation procedure is then employed.
Candidate structures are first screened with the MACE-MP calculator used by the active protocol to discard pathological configurations and rank promising candidates by screened energy. This screening stage can optionally include a short Langevin molecular-dynamics warmup, but the warmup is disabled (\texttt{md\_steps=0}) in the primary CPU benchmark. The best surviving candidate (\texttt{relax\_top\_n=1}) is then fully relaxed and analyzed.
% 第一阶段的candidate生成可以使用这部分内容扩充：The candidate structures undergo a computationally inexpensive screening phase via a preliminary static potential energy evaluation, optionally preceded by a short Langevin molecular-dynamics warmup. This warmup is disabled (\texttt{md\_steps=0}) in the CPU MACE benchmark reported in Section~\ref{subsec:computational_protocol} to reduce runtime; it is retained as an available option for GPU-backed runs. The structures are subsequently ranked, and the most stable candidate is propagated to the full structural relaxation phase.

The post-relaxation analysis reconstructs the actual local coordination environment, detects dissociation and bond changes, and writes structured artifacts that can be consumed directly by the next planning step or by offline benchmark tooling.

% 以下为物理计算的扩充内容
The core physical engine of the \textit{AdsMind} framework relies on machine-learning interatomic potentials (MLIPs), specifically the MACE equivariant message-passing architecture and the MACE-MP foundation model~\cite{batatia2022mace,batatia2023foundation}, functioning as the primary surrogate calculator.

For a given adsorption state, the thermodynamic stability is quantified by the adsorption energy $E_{\text{ads}}$, defined as:
\begin{equation}
    E_{\text{ads}} = E_{\text{total}} - E_{\text{surface}} - E_{\text{adsorbate}}
    \label{eq:adsorption_energy}
\end{equation}
where $E_{\text{total}}$ is the total potential energy of the relaxed adsorbate-slab system, $E_{\text{surface}}$ is the single-point potential energy of the bare surface evaluated with the same cell and calculator settings, and $E_{\text{adsorbate}}$ is the energy of the isolated adsorbate in a vacuum box.

Structural relaxation is performed using the Broyden–Fletcher–Goldfarb–Shanno (BFGS) optimization algorithm. The forces $\mathbf{F}_i$ on each unconstrained atom $i$ are minimized until the maximum force component falls below a predefined threshold $f_{\text{max}}$:
\begin{equation}
    \max_{i} \| \mathbf{F}_i \|_{\infty} \le f_{\text{max}}
\end{equation}

During the optimization, selective constraints are applied. In the standard mode, the bottom fraction of the surface slab atoms (typically one-third of the $z$-axis dimension) is held fixed to simulate the bulk crystal termination, while the upper atomic layers and the adsorbate are fully relaxed.

%% moved from Introduction section
All primary benchmark and matched-baseline experiments use the same MACE-MP-0 small (CPU, float32, dispersion disabled) physical backend so that observed differences reflect search strategy, not energy-function choice; the separate MACE-MP-0 large run is treated only as a force-field sensitivity check.

%% Post-Relaxation Analysis provide feedback

Following physical simulation, the \textbf{Analyzer} systematically parses the relaxed trajectory to extract the precise local coordination environment.
% chemical slip
In AdsMind, \textit{Chemical Slip} is automatically detected when relaxation drives the adsorbate away from the \textbf{Planner}'s intended surface-symbol environment toward a different stable local environment. Canonical site-family mismatch is logged as a related geometric/site slip diagnostic rather than being conflated with the surface-symbol slip flag. 

%Similarly, rearrangement, dissociation, and adsorption failure are all detected and recorded.

% failure/rearrangement/dissociation
% 以下为Gemini扩充的内容，待删改
The connectivity of the relaxed system is established by constructing a localized adjacency matrix $A$. Two atoms $i$ and $j$ are considered bonded if their interatomic distance $d_{ij}$ satisfies:
\begin{equation}
    d_{ij} \le \gamma \left( R_i + R_j \right)
\end{equation}
where $R_i$ and $R_j$ are the natural covalent radii of the atoms. For intramolecular dissociation checks, $\gamma=1.35$ is used to construct the adsorbate subgraph; for adsorbate--surface coordination, the implementation uses $\gamma=1.30$ by default and relaxes it to $\gamma=1.45$ for strongly adsorbed states ($E_{\text{ads}} < -0.5$~eV). Using the adjacency matrix $A$, the system evaluates:
\begin{itemize}
    \item \textbf{Dissociation:} Computed via the connected components of the adsorbate subgraph. If the number of components exceeds one, intramolecular dissociation is flagged.
    \item \textbf{Coordination Environment:} By identifying all surface atoms $k$ bonded to the targeted adsorbate anchor atom $a$ ($A_{ak} = 1$), the actual site type is classified into \textit{ontop} ($\sum_k A_{ak} = 1$), \textit{bridge} ($\sum_k A_{ak} = 2$), or \textit{hollow} ($\sum_k A_{ak} \ge 3$).
\end{itemize}
If the realized site type or surface-symbol fingerprint differs from the planner's intent, the corresponding slip diagnostic is explicitly logged. 

Then the analyzer updates the running memory with the empirical $E_{\text{ads}}$, bond integrity, and slip status, enabling the LLM to dynamically adjust its exploration strategy in subsequent iterations.

\subsection{Ablation Studies and Experimental Setup}

To identify the contribution of different mechanisms in AdsMind's loop search strategy, these ablation switches are designed for experiments. 
% 这部分来自planner会收到的上下文，结合context进行解释消融实验
The \textit{Full} setting enables all three mechanisms. \textit{$-$Slip} removes slip-conditioned guidance and the associated negative constraint. \textit{$-$Forbid} keeps slip detection but suppresses the explicit prohibition against retrying a failed site family. \textit{$-$Term} disables convergence-based early stopping while preserving the rest of the loop. \textit{1-Shot} disables all three mechanisms and sets the maximum attempt budget to one.

% 这部分是实验设置的解释，与前文关系不大，具体的结果在section 3.
All agent-side benchmark runs in this manuscript use frozen experiment configurations rather than mutable product defaults. Four reasoning backends are evaluated: Gemini~2.5~Pro (\texttt{gemini-2.5-pro}), Grok-4 (\texttt{grok-4-0709}), GPT-5.4 (\texttt{gpt-5.4-2026-03-05}), and Claude~Sonnet~4.6 (\texttt{claude-sonnet-4-6}), all run at zero temperature under fixed backend settings. For the primary benchmark protocol, the physical backend is locked to a conservative MACE-MP-0 small configuration loaded through \texttt{mace.calculators.mace\_mp}: \texttt{model=small}, \texttt{default\_dtype=float32}, \texttt{device=cpu}, and dispersion disabled. This choice reflects the practical hardware constraints of the benchmark environment and is held fixed across all reported ablations and backend comparisons. Each frozen configuration records the model identifier and experiment settings needed to reproduce the run.

The original locked CMU statistical package used 15 cases (01--02, 04--05, 09--10, 12--20), and the current manuscript revision extends the CMU ablation audit to the full 20-case manifest. This CMU set spans atomic adsorbates (H), small multi-atom intermediates (OH and NNH), and larger intermediates (CH$_2$CH$_2$OH, OCHCH$_3$, and ONN(CH$_3$)$_2$) across intermetallic and monometallic surfaces. Full runs use a maximum of five attempts. The 1-shot baseline uses the same tooling and physical backend but restricts the planner to a single attempt, allowing us to isolate the value of closed-loop refinement from the value of the base planner alone. Experiments are launched from frozen configurations with fixed seeds and structured result serialization for reproducibility.

For the expanded OCD-GMAE validation reported in the current manuscript revision, we assemble an audited 24-case set from the original locked10 subset (003, 004, 005, 012, 013, 015, 016, 019, 023, 024) plus an extra14 extension (001, 002, 006, 007, 008, 009, 010, 011, 014, 017, 018, 020, 021, 022). The formal expanded OCD24 statistics use all four reasoning backends in the same five-variant matrix. Separately, we evaluate a 50-case representative OCD-GMAE slice (rep50) as a broader Full-versus-1-Shot generalization study across the same four backends. For these expanded OCD24 and rep50 analyses, natural chemistry failures are retained in success-rate and reach-Full statistics, while energy deltas are computed only on runs with valid relaxed adsorption energies.
