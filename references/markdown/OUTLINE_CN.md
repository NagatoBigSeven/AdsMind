# AdsMind：面向自主吸附构型搜索的闭环多智能体框架

**目标期刊**：JCTC

**备选标题**：
- AdsMind：基于闭环LLM-MLFF反馈的自主吸附构型发现
- 超越单次预测：面向表面吸附的自校正多智能体系统

---

## 核心思想

基于LLM的智能体可以提出吸附构型方案，但它们存在**系统性的化学推理偏差**——偏好高对称位点、忽略表面组分效应、且无法从计算结果中学习。

AdsMind实现了闭环：每次MLFF弛豫结果都会反馈给LLM规划器，使系统能够**检测自身错误**（Chemical Slip）、**避免重复犯错**（FORBID约束）、以及**判断何时停止**（自主终止）。

这不是Adsorb-Agent的竞争者，而是一种**互补范式**：他们通过单次推理缩小搜索空间；我们通过迭代物理反馈精炼解。

### 1. 引言——为什么单次LLM推理不够

#### 1.1 吸附构型搜索的重要性

- 多相催化、储能、腐蚀——都始于寻找稳定的吸附质-表面构型
- 组合爆炸：多种位点（ontop/bridge/hollow）x 表面原子 x 吸附质取向 x 构象异构体
- 传统方法：穷举枚举或启发式采样——计算代价高，无法保证全局最优

#### 1.2 LLM驱动方法：潜力与局限

- LLM从训练数据中编码了化学直觉
- Adsorb-Agent（Ock et al., Digital Discovery, 2026）：首个用于吸附的LLM智能体——84%成功率，27%搜索空间缩减
- **但是**：开环架构——LLM提出方案后计算执行，无反馈
- 后果：
  - 解离/异构化的结构被过滤掉，而非从中学习
  - 没有机制检测或纠正系统性偏差
  - 无法区分"计算失败"和"该位点热力学不稳定"

#### 1.3 我们的贡献：闭合回路

AdsMind引入了前人工作中缺失的三个机制：

1. **Chemical Slip检测**——识别弛豫后吸附质实际位点与LLM计划位点不一致的情况，实时诊断LLM推理错误
2. **FORBID约束**——阻止重新探索已被证明不稳定的位点
3. **自主终止**——在检测到收敛时停止探索，无需固定预算

**核心论点**：

> 迭代物理反馈将LLM化学推理从"有依据的猜测"转变为"基于证据的精炼"。由此产生的闭环系统以更少的无效计算找到同等或更低能量的构型。

**关键定位**：与Adsorb-Agent互补而非竞争。他们缩小搜索空间；我们在其中精炼。

### 2. 方法——AdsMind框架

#### 2.1 框架概览（→ **图1**：架构图）

五节点LangGraph状态机：

```
预处理器 → 规划器(LLM) → 方案验证器 → 工具执行器 → 终局分析器
                ↑                              |
                └──────── 反馈回路 ─────────────┘
```

- **预处理器**：通过AutoAdsorbate shrinkwrap算法进行表面分析；识别可用位点类型和表面组成
- **规划器（LLM）**：接收先前所有尝试的完整历史及结果标签；提出下一个构型或终止
- **方案验证器**：在计算前强制执行化学/几何约束（位点-原子对齐、去重）
- **工具执行器**：结构生成（AutoAdsorbate）→ MACE-MP弛豫 → 键完整性分析 → 位点分类
- **终局分析器**：生成包含注意事项的科学报告（能量简并、亚稳态、表面异质性）

#### 2.2 Chemical Slip检测

- **定义**：LLM计划的吸附位点与MLFF弛豫后实际位点之间的不匹配
- 检测方式：比较（计划位点类型, 计划表面原子）vs（实际位点类型, 实际表面原子）
- **物理含义**：计划位点是PES上的鞍点或局部极大值——吸附质滑向邻近极小值
- 每次slip在迭代历史中被标记：`[Unstable Site Warning]`
- LLM学习哪些化学环境对给定吸附质是不稳定的

#### 2.3 FORBID约束

- Chemical Slip发生后，该位点类型+表面原子组合被加入FORBID列表
- 规划器被明确指示："不要再测试{Cu-Pd bridge}类型的位点"
- 验证器拒绝违反FORBID约束的方案
- **效果**：搜索空间随每次迭代缩小，聚焦于化学上有意义的区域

#### 2.4 自主终止逻辑

规划器在以下情况触发 `TERMINATE`：
- 当前最优与先前最优在能量（0.05 eV以内）和几何构型上一致
- 所有主要高对称位点类型已被探索
- 没有剩余的化学推理支持进一步探索
- 达到MAX_RETRIES（5）

#### 2.5 计算细节

**MLFF**：
- MACE-MP-0 medium模型
- 精度：float64（Linux/CUDA），float32（macOS/Apple Silicon）
- 优化器：BFGS，fmax = 0.05 eV/Å（GPU）/ 0.10 eV/Å（CPU）
- slab底部1/3固定；真空层 ≥ 15 Å

**DFT**（验证子集）：
- VASP 6.x，PBE + D3(BJ)色散校正
- ENCUT = 500 eV，k点：6x6x1（Monkhorst-Pack）
- EDIFF = 1E-5 eV，EDIFFG = -0.02 eV/Å
- 策略：以MACE-MP弛豫结构为初猜 → 全DFT弛豫

**LLM**：
- 默认：Gemini 2.5 Pro（生产环境）——选择理由：结构化输出合规性强、单token成本低、初步测试中化学推理能力有竞争力
- Temperature：0.0（确定性）
- SI中进行多模型对比（GPT-5.4、Claude Sonnet/Opus 4.6、Gemini 3.1 Pro/Flash Preview、开源：Qwen 3.5、GLM 5等）——目的是展示闭环架构**无论LLM后端如何**都能提升性能，同时生成跨模型的Chemical Slip统计数据

3-5个代表性表面经全DFT弛豫验证。

| 变化轴 | 体系 | 选择理由 |
|--------|------|---------|
| 单金属基线 | Pt(111) + OH | 研究充分的ORR表面；有丰富DFT文献可交叉验证 |
| 晶面敏感性 | Pt(100) + OH | 相同吸附质不同晶面——测试MACE-MP能否捕捉晶面依赖的排序 |
| 金属间化合物+小吸附质 | Mo3Pd(111) + H | NRR相关；观察到强Chemical Slip——在LLM先验最弱的表面上压力测试MACE-MP |

**验证指标**：
- MACE-MP vs DFT能量排序一致性（目标：≥ 90%）
- 绝对能量误差（目标：< 0.1 eV）
- 结构RMSD（目标：< 0.2 Å）

**核心信息**：MACE-MP能量排序足够可靠以指导搜索，即使绝对能量存在偏差。智能体的作用是找到构型空间的正确区域；DFT确认排序。

### 4. 讨论

#### 4.1 Chemical Slip作为LLM化学推理的代理指标

- Chemical Slip不仅是纠错机制——它揭示了**LLM化学直觉在哪里崩溃**
- 特定表面原子组合上的高slip率表明系统性的训练数据缺口
- **关键洞察**：按表面类型和吸附质分层的Chemical Slip频率可作为**LLM化学推理质量的定量代理指标**——独立于任何下游任务指标
- 跨LLM后端对比（SI-5）：如果模型A在金属间化合物表面上的slip率低于模型B，说明模型A对这些体系有更好的内化化学知识。这比"智能体是否找到了全局最优"更细粒度的评估
- 这一定位将我们系统的副产品转变为**独立的方法学贡献**：一种基于物理的LLM化学知识评估方式，超越了测试事实记忆的文本基准（如ChemBench），转向评估应用推理能力
- 社区潜在用途：其他课题组可以在自己的体系上报告Chemical Slip统计，以在特定领域基准测试LLM化学推理

#### 4.2 闭环 vs 开环：范式差异

- 开环（Adsorb-Agent）：快速、可并行、适合研究充分的表面
- 闭环（AdsMind）：单次迭代较慢，但在复杂表面上找到更好的解
- 二者不互斥——可以组合：Adsorb-Agent提供初始方案 → AdsMind迭代精炼

#### 4.3 MLFF作为代理：何时有效？

- DFT验证表明MACE-MP能量排序总体可靠
- 失效模式：色散主导的结合、强关联体系
- 智能体的价值与MLFF精度正交——它优化的是搜索策略，而非能量函数本身

#### 4.4 局限性

- MACE-MP在某些体系上的精度天花板（如f电子金属）
- LLM API成本和延迟（在SI中量化）
- 当前实现：仅支持平面/台阶表面，不支持缺陷或纳米颗粒
- 每个表面单一吸附质（无共吸附）

### 5. 结论

- AdsMind证明了在LLM推理与物理模拟之间**闭合反馈回路**能显著改善吸附构型搜索
- Chemical Slip检测将LLM错误转化为学习信号
- 与Adsorb-Agent正面对比：结果相当或更优，尤其在复杂表面上
- DFT验证确认MLFF引导的搜索对于构型排序是可靠的
- 开源：代码、结构和日志见GitHub

## 这篇论文为什么能成

1. **贡献明确**：不是"又一个LLM智能体"——而是特定的架构创新（闭环反馈）和特定机制（Chemical Slip）
2. **公平对比**：相同基准、相同表面、与唯一已发表竞争者的直接正面对比
3. **难度递进**：CMU基准（精选）→ OC数据集（多样化）→ DFT验证（真实值）——三层评估体系预防审稿人问"但它能泛化吗？"
4. **物理验证**：DFT确认MLFF引导的搜索有意义，而非仅仅数值自洽
5. **消融实验**：每个组件都有定量验证，而非仅仅描述
6. **双重贡献**：闭环架构是贡献1；Chemical Slip作为LLM化学推理质量的代理指标是贡献2——赋予论文超越特定应用的生命力
7. **互补定位**：将AdsMind与Adsorb-Agent并列而非对立——CMU社区的审稿人不会感到被攻击
